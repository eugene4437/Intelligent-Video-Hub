# Документація статичного аналізу коду

## Використані інструменти
1. **Pylint**: основний лінтер для аналізу якості та стилю (PEP 8).
2. **Mypy**: інструмент для суворої перевірки статичної типізації.
3. **GitHub Actions**: для автоматизації перевірок при кожному push.

## Основні правила
- Довжина рядка обмежена 100 символами.
- Всі функції повинні мати анотації типів аргументів та значень, що повертаються.
- Використання стандарту snake_case для функцій та змінних.

## Запуск перевірок
Локально:
`pylint main.py`
`mypy main.py`
