# Документація з лінтингу коду

## Обраний інструментарій
- **Pylint**: для аналізу якості коду та дотримання стандарту PEP 8.
- **Mypy**: для перевірки статичної типізації.
- **Pre-commit hooks**: для автоматичного запуску перевірок.

## Базові правила
- Максимальна довжина рядка: 100 символів.
- Обов'язкова зміїна нотація (snake_case) для назв функцій та змінних.
- Обов'язкова анотація типів для всіх публічних функцій.

## Інструкція із запуску
1. Запуск лінтера: `pylint src/ main.py`
2. Перевірка типів: `mypy src/ main.py`
